# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the stock Flutter icon with an original "birdsong in colour" Messiaen-evoking icon, wired in as the Linux window icon, a `.desktop` launcher entry, and the Android launcher icon.

**Architecture:** One SVG master (+ a background-less foreground SVG for Android adaptive) is the source of truth; `rsvg-convert` rasterises every needed PNG via `scripts/gen-icons.sh`. The Linux runner loads the bundled 256px PNG and sets it as the GTK window icon; a `.desktop` file + hicolor themed icons are installed by `scripts/install-desktop.sh` (`just install-desktop`); Android icons are generated by `flutter_launcher_icons`.

**Tech Stack:** SVG + `rsvg-convert`; Flutter (asset bundling); GTK/glib C++ (Linux runner); `flutter_launcher_icons` (Android). The artwork is original work released CC0.

**Commands:** `scripts/gen-icons.sh` (raster pipeline); `mise exec -- flutter analyze`; `mise exec -- flutter build linux --debug`; `mise exec -- dart run flutter_launcher_icons`; lint gate `just lint --all` (runs shellcheck/shfmt on the scripts).

**No unit tests:** this feature is artwork + config + a small C++ runner change — there is no Dart logic to unit-test. Each task is verified by building, regenerating, and checking the produced files (dimensions/existence) plus `analyze`/`lint` staying green, and a manual smoke (`just run`).

**Conventions / gotchas:**
- NEVER `git add` the `TODO` file (the user's live scratchpad, shows as `M TODO`) or any `#TODO#`.
- Commit messages: plain imperative (no `feat:`), ending with the trailer `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Stage only the listed files.
- Shell scripts must be shellcheck/shfmt clean (precious lints `scripts/*.sh`).

**Task order:** 1 (masters + raster pipeline) → 2 (Linux window icon) → 3 (.desktop install) → 4 (Android). Tasks 2–4 depend on Task 1's generated PNGs.

- [ ] **Step 3: Create the raster pipeline script**

Create `scripts/gen-icons.sh`:

```bash
#!/usr/bin/env bash
# Rasterise the SVG masters into every PNG the app icon needs. Re-runnable;
# the generated PNGs are committed so a clean checkout has them without
# needing rsvg-convert.
set -euo pipefail

cd "$(dirname "$0")/.."
svg="assets/icon/olivier.svg"
fg="assets/icon/olivier_foreground.svg"
out="assets/icon"

for n in 16 24 32 48 64 128 256 512; do
  mkdir -p "$out/hicolor/${n}x${n}"
  rsvg-convert -w "$n" -h "$n" "$svg" -o "$out/hicolor/${n}x${n}/olivier.png"
done

rsvg-convert -w 256 -h 256 "$svg" -o "$out/olivier_256.png"
rsvg-convert -w 1024 -h 1024 "$svg" -o "$out/olivier_1024.png"
rsvg-convert -w 1024 -h 1024 "$fg" -o "$out/olivier_foreground_1024.png"

echo "Generated app icons under $out"
```

Make it executable: `chmod +x scripts/gen-icons.sh`

- [ ] **Step 3: Set the GTK window icon in the runner**

In `linux/runner/my_application.cc`, inside `my_application_activate`, immediately after the line `gtk_window_set_default_size(window, 1280, 720);`, insert:

```cpp
  // Set the window icon from the bundled PNG so the title bar / taskbar /
  // alt-tab show Olivier's icon. The asset is bundled next to the executable
  // at data/flutter_assets/, so resolve it relative to /proc/self/exe; this
  // works both for `just run` (in-place bundle) and an installed bundle.
  g_autofree gchar* exe_path = g_file_read_link("/proc/self/exe", nullptr);
  if (exe_path != nullptr) {
    g_autofree gchar* exe_dir = g_path_get_dirname(exe_path);
    g_autofree gchar* icon_path =
        g_build_filename(exe_dir, "data", "flutter_assets", "assets", "icon",
                         "olivier_256.png", nullptr);
    g_autoptr(GdkPixbuf) icon = gdk_pixbuf_new_from_file(icon_path, nullptr);
    if (icon != nullptr) {
      gtk_window_set_icon(window, icon);
      gtk_window_set_default_icon(icon);
    }
  }
```

(No new `#include` is needed: `flutter_linux/flutter_linux.h` pulls in `<gtk/gtk.h>`, which provides `GdkPixbuf`, and glib provides `g_file_read_link`/`g_path_get_dirname`/`g_build_filename`.)

- [ ] **Step 2: Create the install script**

Create `scripts/install-desktop.sh`:

```bash
#!/usr/bin/env bash
# Install the Olivier .desktop entry + themed icons into the user's XDG dirs so
# Olivier appears in the application menu / dock. Requires a release bundle
# (build it with: mise exec -- flutter build linux --release). Re-run after a
# rebuild if the bundle path changes.
set -euo pipefail

cd "$(dirname "$0")/.."
repo="$PWD"
bin="$repo/build/linux/x64/release/bundle/olivier"

if [[ ! -x "$bin" ]]; then
  echo "Release bundle not found at: $bin" >&2
  echo "Build it first: mise exec -- flutter build linux --release" >&2
  exit 1
fi

apps="$HOME/.local/share/applications"
icons="$HOME/.local/share/icons/hicolor"
mkdir -p "$apps"

for n in 16 24 32 48 64 128 256 512; do
  dir="$icons/${n}x${n}/apps"
  mkdir -p "$dir"
  cp "$repo/assets/icon/hicolor/${n}x${n}/olivier.png" "$dir/olivier.png"
done
mkdir -p "$icons/scalable/apps"
cp "$repo/assets/icon/olivier.svg" "$icons/scalable/apps/olivier.svg"

sed "s|^Exec=.*|Exec=$bin %U|" "$repo/linux/olivier.desktop" >"$apps/olivier.desktop"

gtk-update-icon-cache -f -t "$icons" 2>/dev/null || true
update-desktop-database "$apps" 2>/dev/null || true

echo "Installed Olivier desktop entry + icons. Look for it in your app menu."
```

Make it executable: `chmod +x scripts/install-desktop.sh`
